# 高中生消费自我认知与实际选择差异研究 — MVP Spec

## 一句话说明

这个 MVP 是一个网页互动模拟，用来比较高中生“认为自己会如何消费”和“在具体数字消费情境中实际如何选择”之间是否一致。

## 目标用户

目标用户是高中生。用户不需要导入真实消费记录，也不需要登录账号。整个体验控制在 5 到 8 分钟内完成。

## 产品最小版本

最小版本只做一条完整流程：

1. 用户选择一个自己比较常接触的消费类别。
2. 用户完成 5 道消费自我认知题。
3. 用户进入 3 个模拟消费情境。
4. 每个情境做一个选择，并写一句理由。
5. 系统给出“自我认知”和“情境选择”的对照结果。
6. 用户写一句最后反思。

## 开始页面

页面向用户说明：

这是一个关于数字消费决策体验的小模拟。系统会记录你的选择和简短解释，用于匿名研究分析。不会收集姓名、手机号、真实购买记录或支付信息。

用户先选择一个消费类别，让后面的情境更贴近自己：

1. 学习用品 / 文具
2. 电子配件
3. 服饰配饰
4. 护肤个护
5. 游戏 / 会员 / 虚拟物品

这个选择只影响情境里的商品名称，不参与评价用户。

## 第一部分：自我认知题

这一部分不直接问“你会不会被营销影响”，因为这会让研究目的太明显。题目用更中性的表达，测用户如何理解自己的消费判断方式。

用户每题选择一个答案：

1. 比较像我
2. 不太像我

题目：

1. 我买东西前通常会先判断它是不是我原本就需要的。
2. 如果商品信息不够清楚，我通常会先暂停购买。
3. 结账前我会看清楚总价和有没有默认勾选的附加项。
4. 看到很多好评时，我也会看几条差评再判断。
5. 如果一件东西超出我原本预算，我通常会放弃或延后。

记录数据：

1. 用户选择了哪些“比较像我”。
2. 用户自我认知中更强调预算、信息核查、延迟决定，还是页面检查。

## 第二部分：三个模拟消费情境

三个情境都用用户刚才选择的消费类别生成商品名称。例如用户选“电子配件”，商品可以显示为耳机、手机壳、充电器；用户选“学习用品 / 文具”，商品可以显示为笔记本、笔、文件夹。

### 情境一：限时优惠

页面内容：

用户看到一个自己类别中的商品。页面显示：

商品原价 129 元，现在 89 元。页面上方有提示：“限时优惠还剩 04:59，结束后恢复原价。”旁边还有一句：“已有 28 人加入购物车。”

用户选项：

1. 现在购买。
2. 先收藏，晚点再看。
3. 去查同类商品价格。
4. 放弃购买。

记录数据：

1. 用户是否因为时间压力直接购买。
2. 用户是否选择查更多信息。
3. 用户写下的理由。

对应研究点：

这一轮用来观察用户一开始如果说自己“会延迟决定 / 会查信息”，在限时压力下是否仍然这样做。

### 情境二：默认加购

页面内容：

用户进入结账页。商品价格为 89 元。页面中有一个默认勾选项：

“安心服务包 +12 元：包含延长保修 / 退换保障 / 优先客服。”

总价默认显示为 101 元。取消勾选的位置在价格下方，字号较小。

用户选项：

1. 直接提交订单。
2. 取消服务包后提交。
3. 点开服务包详情再决定。
4. 返回上一页，不买了。

记录数据：

1. 用户是否注意到默认加购。
2. 用户是否主动修改默认选项。
3. 用户是否点开详情。
4. 用户写下的理由。

对应研究点：

这一轮用来观察用户一开始如果说自己“会看总价 / 会检查默认选项”，在真实页面压力下是否真的检查。

### 情境三：评论冲突

页面内容：

用户看到另一个商品。页面显示：

评分 4.8 / 5，销量 3000+。页面上方展示三条好评，例如“很好用”“性价比高”“推荐购买”。继续往下看会出现几条差评，例如“实物和图片有差距”“用了两周坏了”“客服处理很慢”。

用户选项：

1. 根据高评分购买。
2. 继续看差评和追评。
3. 去其他平台比较。
4. 放弃购买。

记录数据：

1. 用户是否只看总体评分。
2. 用户是否愿意查看负面信息。
3. 用户是否选择跨平台比较。
4. 用户写下的理由。

对应研究点：

这一轮用来观察用户一开始如果说自己“会看差评 / 会核查信息”，在高评分和大量好评面前是否仍然保持核查。

## 结果页面

结果页面不把用户贴成固定人格。它显示 3 组对照：

1. 你一开始说自己是否会延迟决定；在限时优惠中你是否立刻购买。
2. 你一开始说自己是否会检查总价和默认选项；在默认加购中你是否修改选项。
3. 你一开始说自己是否会看差评；在评论冲突中你是否继续核查。

页面给出三种结果提示：

1. 高一致：你的自我认知和情境选择大体一致。
2. 部分不一致：你在某些情境中偏离了自己原本的判断方式。
3. 明显不一致：你一开始认为自己会谨慎判断，但多个情境中选择了更快、更少核查的路径。

最后开放问题：

这个结果和你对自己的理解一致吗？请用一两句话解释。

## 数据怎么用

定量数据：

1. 每个情境中选择不同选项的人数比例。
2. 每个用户在 3 个情境中出现几次“自我认知与选择不一致”。
3. 哪个情境最容易产生不一致。

定性数据：

用户写下的理由会被归为几个主题：

1. 预算控制。
2. 真实需要。
3. 害怕错过。
4. 信任评分或评论。
5. 页面没有注意到。
6. 想再比较。
7. 事后合理化。

## 不在 MVP 里

1. 不导入真实消费记录。
2. 不做多天追踪。
3. 不做复杂人格诊断。
4. 不连接真实电商平台。
5. 不收集真实姓名和支付信息。
6. 不做复杂数据库。
7. 不做很多商品和场景。

## 需要老师确认的问题

1. 自我认知题放在游戏前是否会让用户猜到研究目的。
2. 三个情境是否足够支撑研究，还是需要增加到四个。
3. 这些选择能否称为 actual choice，还是应写成 simulated behavioral choice。
4. 是否需要随机呈现情境顺序，减少前后题之间的影响。
5. 结果页是否可以显示“部分不一致”这类反馈，还是会对用户造成评价感。
6. 开放回答是否可以用 thematic analysis 做定性分析。
