# 高中生消费自我认知与实际选择差异研究 — MVP Spec

## 它做什么

这个 MVP 是一个网页互动模拟：用户先完成简短的消费自我认知判断，再进入 3 个数字消费情境，最后看到自己的自我认知和实际选择之间是否出现差异。

## 给谁用

目标用户是高中生，尤其是经常接触短视频、电商平台、社交媒体推荐和线上娱乐消费的学生。

## 核心流程

1. 用户进入网页，看到项目说明和匿名数据提示。
2. 用户完成 5 道中性的自我认知题，内容围绕预算意识、延迟决定、信息核查、默认选项检查和情绪影响。
3. 用户进入 3 个模拟消费情境，每一轮从 2 到 4 个选项中做出选择，并写一句选择理由。
4. 系统把用户的自我认知和情境选择进行对照，显示几个具体差异点。
5. 用户回答最后一个开放问题：这个结果和你对自己的理解一致吗？为什么？

## 不在 MVP 里

1. 不导入真实消费记录。
2. 不做多日追踪实验。
3. 不做复杂预算系统。
4. 不做完整人格测试或固定消费人格诊断。
5. 不接入真实支付、电商账号或个人身份信息。
6. 不做大型数据库后台，MVP 阶段可以先用本地导出或表单收集。

## 怎么收集反馈

MVP 收集四类数据：

1. 自我认知选择：用户在开始时如何评价自己的消费判断方式。
2. 情境选择：用户在每个消费情境里选择了什么。
3. 选择理由：用户用一句话解释自己为什么这样选。
4. 结果反应：用户是否认为结果符合自己对自己的理解。

## 最小情境设计

第一轮：限时优惠。页面显示一件用户可能感兴趣的普通商品，并出现限时折扣提示。观察用户是否立刻购买、延迟决定或查更多信息。

第二轮：默认加购。页面默认勾选附加服务或套装。观察用户是否会主动检查并取消默认选项。

第三轮：评论冲突。页面同时出现高评分和几条具体差评。观察用户是否只看总体评分，还是愿意继续核查信息。

## 数据分析方式

定量部分只做简单统计：每个情境中不同选择的人数比例，以及自我认知和实际选择一致/不一致的比例。

定性部分使用主题分析：把用户的解释归为几个主题，例如预算控制、真实需要、害怕错过、信任评价、界面忽略、事后合理化。
