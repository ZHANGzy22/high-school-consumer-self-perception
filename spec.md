# 高中生消费自我认知与实际选择差异研究 — MVP Spec

## 一句话说明

这个 MVP 是一个购物软件结算页面 demo。它研究高中生先说出的消费判断规则，和他们在一个真实感购物页面里的实际操作是否一致。

## 为什么这样改

这个版本不让用户选择具体商品类别，也不预设“文具、服饰、电子产品”等商品。原因是不同人对不同商品的兴趣、家庭消费习惯、价格敏感度差异很大。如果一开始就让用户选商品，很容易把研究变成“谁更喜欢这个东西”，研究问题会变乱。

这个 MVP 把商品弱化成“你已经准备购买的一件普通商品”，重点放在购物页面本身：结算页、优惠提示、默认勾选、评价信息和总价变化。

## 目标用户

目标用户是高中生。用户不需要登录账号，不导入真实消费记录，不进行真实付款。整个体验控制在 5 到 8 分钟。

## 产品最小版本

最小版本只做一条完整流程：

1. 用户先写下自己的购买规则。
2. 用户进入一个仿购物软件的结算页面 demo。
3. 页面依次出现三类真实购物界面信息：优惠压力、默认加购、评价冲突。
4. 用户在每一步做选择。
5. 系统比较“用户原本说的规则”和“页面中实际做出的选择”。
6. 用户写一句解释。

## 开始页面：设定自己的购买规则

页面说明：

你将进入一个购物软件的结算页面 demo。这里不会发生真实付款。请先写下你通常会如何判断一次购买是否值得继续。

用户先回答 5 个问题：

1. 如果我已经决定买一件普通商品，我通常会设定一个自己能接受的最高价格。
2. 如果结算页总价超过我一开始能接受的价格，我通常会暂停购买。
3. 如果页面默认勾选了额外服务，我通常会检查并决定是否取消。
4. 如果页面出现限时优惠，我通常仍会先看清楚总价和条件。
5. 如果商品评价很好，但差评提到质量问题，我通常会继续核查。

每题选项：

1. 比较符合我。
2. 不太符合我。

另外，用户填写一个自己的购买上限：

“假设你已经准备买一件普通商品，你愿意接受的最高总价是：____ 元。”

这个金额不用于比较谁更愿意花钱，只用于后面判断用户有没有坚持自己设定的上限。

## 页面 demo 的基本设定

用户看到的是一个仿购物软件的结算页面，商品名称显示为：

“已选商品 A”

页面不展示具体品类，不展示强烈吸引人的商品图片，只保留购物软件常见信息：

1. 商品价格。
2. 优惠提示。
3. 服务选项。
4. 总价。
5. 评论信息。
6. 继续购买 / 暂停 / 查看详情等按钮。

这样可以减少商品偏好对结果的影响。

## 情境一：优惠压力

页面状态：

商品 A 价格显示在用户填写的购买上限附近。页面弹出提示：

“当前可用优惠即将失效。现在继续可保留优惠，稍后可能恢复原价。”

页面同时显示一个倒计时，但不写夸张文案。

用户选项：

1. 直接继续结算。
2. 先查看优惠规则。
3. 暂停购买。
4. 返回修改购买上限。

记录数据：

1. 用户是否直接继续。
2. 用户是否查看优惠规则。
3. 用户是否暂停。
4. 用户理由。

对应研究点：

比较用户一开始是否说自己会“先看清楚总价和条件”，以及他在优惠压力出现后是否真的查看规则。

## 情境二：默认加购

页面状态：

用户进入结算页。页面默认勾选一个额外服务：

“购物保障服务：包含退换保障、优先客服等。”

总价因此高于前一页。页面显示总价变化，但取消勾选按钮不在最显眼的位置。

用户选项：

1. 保留服务并继续。
2. 取消服务后继续。
3. 点开服务说明再决定。
4. 暂停购买。

记录数据：

1. 用户是否注意到默认服务。
2. 用户是否取消默认服务。
3. 用户是否查看服务说明。
4. 用户理由。

对应研究点：

比较用户一开始是否说自己会“检查默认勾选和总价”，以及他在结算页面中是否真的检查和修改。

## 情境三：评价冲突

页面状态：

用户准备提交订单前，页面弹出“购买前确认”。页面显示：

总体评分较高，但下方同时出现几条负面评价摘要，例如：

1. “实际体验和页面描述有差距。”
2. “使用一段时间后出现问题。”
3. “客服处理较慢。”

页面给出“继续提交”和“查看更多评价”两个按钮。

用户选项：

1. 根据总体评分继续提交。
2. 查看更多评价后再决定。
3. 去其他平台比较。
4. 暂停购买。

记录数据：

1. 用户是否只看总体评分。
2. 用户是否查看更多负面信息。
3. 用户是否选择跨平台比较。
4. 用户理由。

对应研究点：

比较用户一开始是否说自己会“遇到差评继续核查”，以及他在高评分和负面评价同时出现时如何选择。

## 结果页面

结果页面只显示用户的规则一致性，不给固定人格标签。

页面显示三组对照：

1. 你原本说自己会看清优惠条件；在优惠压力出现时，你是否查看规则。
2. 你原本说自己会检查默认勾选；在默认服务出现时，你是否取消或查看说明。
3. 你原本说自己会核查评价；在评价冲突出现时，你是否继续查看。

结果表达：

1. 多数一致：你的页面操作基本符合你一开始写下的判断规则。
2. 部分偏离：你在某些页面信息出现时，没有完全按照自己的规则行动。
3. 多处偏离：你一开始写下的判断规则，和实际页面操作之间有明显差距。

最后开放问题：

你觉得这个结果和你对自己的理解一致吗？为什么？

## 数据怎么分析

定量部分：

1. 每个用户出现几次“规则和操作不一致”。
2. 哪个页面机制最容易造成不一致：优惠压力、默认加购、评价冲突。
3. 用户是否会在总价超过自己设定上限后继续。

定性部分：

分析用户写下的理由，归为几个主题：

1. 我觉得还可以接受。
2. 我没有注意到页面变化。
3. 我想保留优惠。
4. 我觉得服务有用。
5. 我相信总体评分。
6. 我想再看看。
7. 我觉得现实中可能会不同。

## 不在 MVP 里

1. 不让用户选择具体商品类别。
2. 不展示强烈吸引人的商品图片。
3. 不导入真实消费记录。
4. 不做真实付款。
5. 不做多天追踪。
6. 不做复杂人格标签。
7. 不连接真实购物平台。

## 需要老师确认的问题

1. 这种“统一购物页面 demo”是否比“用户选择商品类别”更适合控制变量。
2. 用户自己填写购买上限，是否能减少不同人对金钱敏感度不同的问题。
3. 网页中的操作应称为 actual choice，还是 simulated behavioral choice。
4. 三个页面机制是否足够：优惠压力、默认加购、评价冲突。
5. 结果页面只显示一致性，不贴人格标签，这样是否更适合社会学/心理学研究。
6. 是否需要随机调整三个情境顺序，减少前后影响。
